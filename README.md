# 나니 분기 스튜디오 (Nani Branch Studio)

비주얼노벨 분기를 **블록(#라벨) + 선택지**로 설계하면, **분기도(흐름도)** 와 **Naninovel 1.18.1 문법의 `.nani` 코드**를 자동 생성해 주는 정적 웹 도구.
수업용 — 서버 없음, 설치 없음, API 키 없음. `index.html` 파일 하나가 전부입니다.

## 사용법 (학생용)

1. `index.html`을 브라우저로 열기 (더블클릭)
2. **캔버스에 블록 배치** — 블록 하나 = `# 라벨` 하나 (블루프린트 노드처럼)
   - 제목줄 드래그로 이동 · "뭐 할지" 한 줄 메모 · 선택지에 **한글 문구** 입력
3. **포트 연결** — 선택지 오른쪽 ○를 잡고 다음 블록 위에 드롭 = `goto` 연결
   - 연결 해제: 연결선 클릭 · 선택지 없는 블록의 ○ = `@goto` (비우면 엔딩)
4. 아래 코드 창에서 실시간 변환 확인 → `.nani` 다운로드 → 유니티에 넣기
5. **생성된 코드의 `; ▼ 여기에 …` 주석 자리에 각자 배경·캐릭터·대사를 채워 넣기** ← 개별 과제

## 블록 → 코드 변환 규칙 (Naninovel 1.18.1)

| 블록에서 | 생성되는 코드 |
|---|---|
| 블록 하나 | `# 라벨` + `; ▶ 메모` + 대사 채울 자리 주석 |
| 선택지 (한글 문구 그대로 보존) | `@choice "문구" set:... goto:.라벨` × N + **`@stop` 자동 추가** |
| 선택지 없음 + 다음 블록 지정 | `@goto .라벨` |
| 선택지 없음 + 다음 없음 (엔딩) | `@stop` 자동 추가 |
| 조건 판정 블록 | `@if` / `@elseif` / `@else` + `@goto` + **`@endif` 자동 닫기** |
| 선택지의 set에 쓴 변수 | 스크립트 맨 위에 `@set 변수=0` 자동 초기화 |

`@stop` 누락, `@endif` 누락 같은 수업에서 가장 흔한 실수를 도구가 원천 차단합니다.

## 배포

GitHub Pages: 저장소 Settings → Pages → Branch `main` / root 선택 → 생성된 주소를 학생들에게 공유.
작업물은 각자 브라우저(localStorage)에 자동 저장됩니다.
